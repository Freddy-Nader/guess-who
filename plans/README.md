## Findings considered and rejected

- **Dead `car03_Click`/`car33_Click` handlers**: not worth a standalone fix; deleted as part of plan 006's menu conversion.
- **`Byte` overflow on `puntacion`** (score caps at 255 questions): unreachable in any real game; not worth code.
- **Relative `Picture["img/…"]` paths**: correct Gambas convention (project-relative), not a bug.
- **Dead `Repeat/Until` guards in `asignacion()`** (e.g. `Until r1 <> 3` after `Int(Rnd(1, 3))` which can never yield 3): harmless; removed incidentally by plans 002/005 where they rewrite those loops, not worth touching elsewhere.
- **`Inicio.class` calling `Plantilla.Form_Open()` directly** (double-open on first show): pre-existing turn-switch mechanism, behavior-preserving to leave; noted as a constraint in plan 006.
- **Accent fixes on logic-compared strings** (e.g. `"Cafe"` eye color): requires changing assignment + comparison in lockstep for zero user-visible gain (option strings barely surface in UI); deferred until/unless i18n work happens.
